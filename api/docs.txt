Описание Asian Wave API v0.5

api.json:

	radio-v2 - данные о состоянии всех потоков радио
	  jp / ru /kp
	    online - "1" если радио онлайн, "0" если оффлайн
	    rj - текущий RJ в эфире ("Auto-DJ" означает, что вещание ведёт автодиджей)
	    song - массив с данными о треках
	      curr - играющий сейчас в эфире
	      next - следующий
	      prev - массив с семью предыдущими треками
	    listeners - количество слушателей

	anime - данные о состоянии стрима на /anime
	  формата ответа пока нет

	timestamp - время последнего обновления данных (происходит раз в минуту)

	В случае, если данные получить невозможно, "radio-v2" и/или "anime" принимают значение null.


anime-sched.json / radio-sched.json:

	Данные для каждого элемента расписания представлены в следующем виде:

	s - таймштам начала
	e - таймштамп конца
	title - название
	link - ссылка (может не быть)
	secret - если равен "true", элемент необходимо не показывать в расписании (может не быть)


noti.json:

	Оповещение одно для всех разделов и имеет следующий вид:
	
	enabled - "true" если включено и надо показывать, "false" если выключено
	time - таймштамп создания оповещения
	text - тест оповещения (могут присутствовать HTML-теги)
	color - цвет фона оповещения (может не быть)

vk-info.json:

	com - данные о сообществе
	  id - ID сообщества
	  url - короткий адрес
	  pic - миниатюра аватара

	posts - данные о последних пяти постах
	  id - id поста
	  time - таймштамп времени публикации поста
	  type - тип поста (пост или репост)
	  pin - закреплён пост в сообществе или нет
	  text - содержание поста
	  pic - опциональная картинка
	    small - маленький размер (отдаётся через прокси)
		big - большой размер (но не оригинал)


vk-stream.json:

	url - адрес iframe стрима
	pic - картинка
	viewers - количество зрителей (в ВК)
	quality - качество стрима
	timestamp - время последнего обновления данных (происходит раз в минуту)
	
	viewers и quality могут иметь значение null, если стрим ещё не начался
