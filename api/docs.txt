Описание Asian Wave API v0.4.3

api.json:

	radio-v2 - данные о состоянии всех потоков радио
	  jp / ru /kp
	    online - "1" если радио онлайн, "0" если оффлайн
	    rj - текущий RJ в эфире ("Auto-DJ" означает, что вещание ведёт автодиджей)
	    song - массив с данными о треках
	      curr - играющий сейчас в эфире
	      next - следующий
	      prev - массив с семью предыдущими треками
	    listeners - количество слушателей

	anime - данные о состоянии стрима на /radio
	  online - "1" если радио онлайн, "0" если оффлайн
	  viewers - количество смотрящих

	timestamp - время последнего обновления данных (происходит раз в минуту)

	В случае, если данные получить невозможно, "radio-v2" и/или "stream" принимают значение null.


streams-shed.json:

	Данные для каждого стрима представлены в следующем виде:
	[
	  таймштамп начала стрима,
	  таймштамп конца стрима,
	  название стрима,
	  ссылка на пояснение
	]

	Ссылка на пояснение опциональна.


noti.json:

	Каждое оповещение имеет следующую структуру:
	[
	  текст оповещения,
	  таймштамп создания оповещения
	]

	В тексте оповещения могут присутствовать html-теги.
	В случае, если оповещение отключено, то оба поля будут иметь значение null


vk-info.json:

	com - данные о сообществе
	  id - ID сообщества
	  url - короткий адрес
	  pic - миниатюра аватара

	posts - данные о последних пяти постах
	  id - id поста
	  time - таймштамп времени публикации поста
	  type - тип поста (пост или репост)
	  ad - рекламный пост (1) или нет (0)
	  text - содержание поста
	  photo - опциональная картинка
	    small - маленький размер (отдаётся через прокси)
		big - большой размер (но не оригинал)


vk-stream.json:

	url - адрес iframe стрима
	pic - картинка
	viewers - количество зрителей (в ВК)
	quality - качество стрима
	timestamp - время последнего обновления данных (происходит раз в минуту)
	
	viewers и quality могут иметь значение null, если стрим ещё не начался
